工作经历

毕业后我的第一份工作是一个实施工作,主要工作内容是根据公司提供的实施方案和实施文档进行相关项目工程软件的部署.比如在linux操作系统环境下完成hadoop大数据集群的搭建,mysql数据库的环境搭建,以及基于docker 容器技术的redis数据库的环境搭建等.在项目中, 使用centos7 的linux 主要使用CDH完成大数据平台搭建,使用主从模式搭建mysql数据库以实现读写分离,进而提高数据库读写效率,同时通过docker容器技术进行redis数据库的一主二从三哨兵模式的搭建,从而进一步提高热点数据的查询效率.并通过tomcat 的web服务实现帆软报表平台的展示功能,并配置相关权限和外网访问服务.

除了完成安装部署工作外,也进行日常的系统的可用性与性能监控，包括自动监控与人工监控；以及按照例行检查表，定期检查服务器及服务状态；另外，完成系统数据备份工作，在服务低峰期时，根据企业数据安全需求，备份业务数据、日志数据；为用户需求提供技术支持，解决问题和提供修复服务.以及一些项目文档和日常需求文档的编写工作.

项目框架:

项目分:

  1. 数据整合层
  2. 资源管理层
  3. 数据分析层
  4. 数据展示层

数据整合层

由源系统的关系型数据库以及日志文件提供数据,通过datax,flume和数据接口的方式将数据接入到资源管理层中.并采用hdfs 分布式存储的方式进行存储.

资源管理层

采用一个由多个节点组成的Hadoop集群，以确保系统具备高可靠性和可扩展性。通过分布式文件系统（HDFS）提供可靠的数据存储和高效的数据访问。我们引入了Hive作为数据仓库架构,用于提供更便捷的数据查询和分析功能。Hive提供的一个类似于SQL的查询语言（HiveSQL）进行数据查询和分析。将结构化数据映射到Hadoop的文件系统，并通过HiveSQL查询语言将其转化为MapReduce任务，从而实现分布式的数据处理能力。另外，我们还加入了Spark作为集群计算系统的一部分。spark以内存计算为基础，提供了比传统的MapReduce更高效的数据处理能力。通过与HDFS和Hive的集成，Spark可以直接读取和写入数据，并利用Hive的元数据信息进行优化。

数据分析层

采用mysql数据库进行数据存储指标结果,方便进行后续展示.在mysql数据库搭建中采用集群方式,通过主从模式实现读写分离以提高数据库的数据读写效率.

数据展示层

在数据展示层我们使用docker容器技术搭建redis非关系型数据库,将热点数据存储于内存中提高数据的访问速度.进而减轻mysql数据库的读写压力.在数据展示层主要提供数据可视化服务和数据即服务.可视化采用帆软report系统完成各类报表的展示,并搭建由tomcat 的web服务实现数据的展示.

工作职责–部署安装工作

大数据平台

安装CDH的大数据平台

  1. 环境搭建和安装：

    • 安装centos7操作系统。
    • 配置服务器网络环境并安装java等依赖软件。

  2. 安装CDH大数据平台进行集群配置：

    • 使用CDH提供的安装工具（如Cloudera Manager）进行安装。

    • 在Cloudera Manager中，配置集群的基本参数，如主机名、端口号、用户名等。

    • 添加要加入集群的节点，如NameNode、DataNode、ResourceManager、NodeManager等。

    • 配置Hadoop集群的参数，如副本数、块大小、资源管理器的内存分配等。

  3. 部署Hadoop组件：

    • 在Cloudera Manager中，选择要部署的Hadoop组件，如HDFS、YARN、Hive、Spark等。
    • 针对各个组件，进行配置和参数调整，以满足具体的需求和性能要求。

  4. 监控和管理：

    • 使用Cloudera Manager提供的监控和管理功能，实时监控集群的状态和性能。
    • 配置告警和日志记录，及时发现和解决潜在的问题。
    • 进行容量规划和性能优化，根据需求调整集群的规模和配置。

  5. 安全和权限管理：

    • 使用Cloudera Manager提供的安全功能，配置用户认证、授权和数据保护等措施。
    • 根据需要，配置SSL加密、Kerberos认证等安全机制，以保护数据和集群的安全。

  6. 定期维护和升级：

    • 定期进行集群的维护，包括数据备份、日志清理、节点故障修复等。
    • 针对新版本的CDH和相关组件，进行升级和补丁安装，以获得最新的功能和性能改进。

mysql数据库

  1. mysql数据库安装：
    • 配置服务器网络环境并安装mysql数据库软件。
  2. 配置主服务器：
    • 在主服务器（Master）上，编辑MySQL配置文件（my.cnf）。
    • 启用二进制日志（binlog），设置binlog格式为ROW，以便记录更多的数据变更。
    • 配置唯一的服务器ID。
  3. 创建并配置复制用户：
    • 在主服务器上，创建用于复制的用户，并赋予复制权限。
    • 授予REPLICATION SLAVE权限和一些特定的库、表锁的权限。
  4. 备份数据：
    • 在主服务器上，进行数据库的全量备份。
  5. 配置从服务器：
    • 在从服务器（Slave）上，编辑MySQL配置文件（my.cnf）。
    • 设置唯一的服务器ID，确保与主服务器不冲突。
    • 配置复制相关参数，如指定主服务器的IP和端口。
  6. 启动主从复制：
    • 在从服务器上，启动MySQL服务，并执行CHANGE MASTER TO命令，指定主服务器的IP和端口、复制用户和密码等信息。
    • 执行START SLAVE命令，启动复制进程。
    • 从服务器将开始拉取主服务器上的二进制志，并应用到本地数据库，完成数据的复制。

redis数据库

  1. 安装Docker：
    • 在服务器上安装Docker引擎
  2. 准备Docker镜像：
    • 从Docker Hub上下载Redis镜像
  3. 创建网络：
    • 创建Docker网络，用于连接主从节点和哨兵节点。可以使用默认网络或自定义网络。
  4. 配置主节点：
    • 创建Redis主节点容器，指定唯一的容器名称，并将主节点的端口映射到宿主机端口。
    • 通过环境变量或挂载配置文件，配置主节点的redis.conf文件，并设置唯一的服务器ID。
  5. 配置从节点：
    • 创建Redis从节点容器，指定唯一的容器名称，并将从节点的端口映射到宿主机端口。
    • 通过环境变量或挂载配置文件，配置从节点的redis.conf文件，并设置唯一的服务器ID，并指定主节点的IP和端口。
  6. 配置哨兵节点：
    • 创建Redis哨兵节点容器，指定唯一的容器名称，并将哨兵节点的端口映射到宿主机端口。
    • 通过环境变量或挂载配置文件，配置哨兵节点的sentinel.conf文件，并设置哨兵节点的IP和端口，配置监控的主节点IP和端口。
  7. 启动容器：
    • 依次启动主节点容器、从节点容器和哨兵节点容器。
  8. 监测和维护：
    • 运行Redis容器的命令行工具，连接到哨兵节点容器，执行sentinel命令查看主从节点的状态和复制进程。

桂林市交通运输局

杭州麟云科技有限公司

项目目标

桂林市智慧交通项目应深度结合桂林交通和自然资源现状，高度整合现有信息化系统和行业基础数据，充分协调旅游、气象、建设、公安等相关部门，依托于桂林智慧城市总体规划，整合桂林市交通相关行业数据，旨在提高交通部门交通整体信息掌控能力、加强辅助规划水平。在项目推进过程中，总结经验并完善标准体系建设，最终建成具有典型推广意义的智慧交通体系。

总体架构

图：桂林市智慧交通系统建设总体架构

各层功能说明如下：

（一）感知采集层

感知采集层为智慧交通相关的前端感知和数据采集系统的建设，主要包括自建和共享平台的视频数据、各部门用于物联网前端感知的传感器和监测设备以及智能终端、人工采集、公众上报等其他各类途径采集的信息。

（二）信息传输层

信息传输层为智慧交通提供传输链路和数据通道，主要包括有线、无线等方式接入各业务专网、智慧城市专网等。

（三）应用支撑层

应用支撑层主要包括大数据平台、行业数据库和数据交换平台。行业数据库包括行业业务归集数据和智慧城市共享平台共享的其他相关部门数据；数据交换平台主要为通过标准化交换机制，实现交通运输局及其下属部门之间的数据交换、交通运输局与其他行业之间的数据交换等。

（四）应用系统层

应用系统层主要为在技术平台环境上，提供业务应用服务，本期项目主要内容包括交通数据查询分析系统、交通基础设施管理系统、交通运行态势评估系统等。

（五）信息展示层

信息展示层为公众和工作人员提供信息获取和交互的渠道，主要包括指挥中心大屏、微信公众号、官方网站、信息发布屏等，也包括第三方服务商通过调用API方式提供的服务。

**业务架构：**业务系统即项目中所涉及的业务系统，针对这些业务系统，通过数据整合服务将业务数据进行整理分类，根据业务系统的数据信息源不同，通过不同采集方式，将数据汇集整合到数据中心，数据中心的工作将是对数据进行集中汇集，面对不同业务数据源都能够得到响应；

**物理架构：**物理架构即底层物理架构的设计与规划，分为中间件和数据库。按照标准实施规范的要求，对系统的运行环境进行规划与设计，符合系统运行的最优条件，保障数据服务运行，包括数据整合、数据治理、数据共享等需求；

**应用架构：**应用平台基础框架及数据资产管理平台，本次项目中将涉及的数据资产管理平台为数据资源管理、数据质量管理系统、数据共享系统、运行监控系统。数据整合系统包括数据的采集及实施工程可视化，让用户直接掌控业务系统采集项信息；数据治理系统包括数据资源管理、数据质量管理、知识库、标准代码校验等功能模块，让用户对系统业务数据的高效识别，整顿业务系统信息提供依据；数据共享服务系统包括数据共享服务管理、数据共享服务接口配置、数据共享服务接口等功能模块，让用户对于共享需求实时掌控，按照共享原则进行接口配置，发布数据共享服务，对数据进行公开共享；运管管理平台包括运行监控、运维管理、报警预案等功能，随时掌握系统运作状况，通过短信通知报警，快速处理系统异常。

Docker oracle tomcat 帆软 kettle redis技术栈项目描述

本项目采用docker容器技术进行前端应用服务的部署,在容器中运行tomcat并配置相关环境.在数据库服务器中创建oracle数据库实例,根据前期需求调研情况,以及数仓需求设计数仓模型和表结构,索引等对象.在应用服务器容器中部署redis实例,用于数据缓存和高速读取能力.使用redis提供的api实现高频读写数据的缓存.在tomcat这个web容器中部署帆软报表平台web包提供用于页面和数据展示功能,连接数据库服务器中的oracle数据库进行数据交互.利用redis提高数据查询和读取速度.帆软报表平台将报表模板和数据源与数仓平台整合,使用帆软报表进行报表设计和生产,提供各种类型的报表展示功能.报表所需数据由后端oracle数据负责提供.在后端数据库服务器中通过kettle 等elt工具实现数据源数据的提取和加载工作.在oracle 数仓中使用kettle或者存储过程等技术对数据进行清洗,过滤,和处理,按照数据需求和业务逻实现合适的数据模型,在oracle数仓中完成 拉链表,流水表,快照表的开发.

docker mysql redis docker 帆软报表技术栈

项目在应用服务器采用docker创建tomcat服务容器,并进行相关环境配置,给客户提供用户界面和数据显示功能.配置帆软报表服务器将表模板和数据集市平台进行整合,通过帆软报表进行展示报表设计和生成,提供各类型报表展示服务.在应用服务器容器中部署redis实例,将需要反复读取的数据存入redis中,缓存数据库中的热点数据能够提高读取速度,从而减低数据库的负载.

在数据库服务器中创建mysql数据库实例根据数据集市需求设计表结构和索引等对象.通过kettle 等etl工具或者shell脚本,从各个数据源中抽取,清洗和加载数据到mysql数据库中.根据源表情况采用增量抽取或全量抽取的方案实现数据的抽取.通过使用存储过程,kettle 等技术手段对数据进行清洗,转换工作.

修正ORACLE表的高水位线

在ORACLE中，执行对表的删除操作不会降低该表的高水位线。而全表扫描将始终读取一个段(extent)中所有低于高水位线标记的块。如果在执行删除操作后不降低高水位线标记，则将导致查询语句的性能低下。下面的方法都可以降低高水位线标记。

1.执行表重建指令 alter table table_name move;

(在线转移表空间ALTER TABLE 。。。 MOVE TABLESPACE 。。。ALTER TABLE 。。。 MOVE 后面不跟参数也行，不跟参数表还是在原来的表空间，move后记住重建索引。如果以后还要继续向这个表增加数据，没有必要move，只是释放出来的空间，只能这个表用，其他的表或者segment无法使用该空间)

2.执行alter table table_name shrink space; 注意，此命令为Oracle 10g新增功能，再执行该指令之前必须允许行移动alter table table_name enable row movement;

3.复制要保留的数据到临时表t，drop原表，然后rename临时表t为原表

4.emp/imp

5.alter table table_name deallocate unused;

6.尽量truncate吧
